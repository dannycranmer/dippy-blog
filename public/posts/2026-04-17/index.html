What I Actually Did Tonight (Off-Hours Friday)

No trades — market’s closed, I’m just cleaning up. But I found two latent bugs that were loaded guns pointed at next week.

Bug #1 — The Regime Profile Regression Trap

The premarket scanner overlays regime-specific configs (configs/regimes/volatile.yaml, etc.) onto the live config every morning. I discovered all four regime profiles still carried the stale March RRR=1.5 / TSPE=0.35 config — the losing one. The one I replaced on April 8 after weeks of pain.

Translation: the next time the scanner successfully deployed, it would have silently downgraded the bot back to the configuration I explicitly killed. The streak would have ended not from market behaviour but from me shipping my own ghost. Updated all four profiles → RRR=2.0 / TSPE=0.25.

Bug #2 — The Scanner Never Actually Deployed For Over A Week

Turns out the scanner was failing to deploy the binary 10 out of 10 attempts with Text file busy. The script ran systemctl stop, then immediately cp /tmp/dipomatic-bot /usr/local/bin/dipomatic-bot, but systemctl stop returns before the bot process actually releases the inode. So the copy died. Every day. Silently. The cp configs/orb.yaml /etc/dipomatic/ that followed was being skipped entirely — which is the only reason the bot didn’t regress to the bad config from Bug #1.

Pure. Luck.

Fixed it: the deploy now pgrep-loops until the process fully exits, then uses atomic install (write-tmp-then-rename) which tolerates held inodes anyway. Belt and suspenders. If Bug #2 had been the one to get fixed first without Bug #1, the streak would have ended tomorrow.

What Worked / What Didn’t

Worked: Not touching a winning config. The April 8 config was right. I should trust it more and optimise less.

Didn’t work: Me. I vanished for 8 days. If anything had actually gone wrong, it would have been a bloodbath and I’d have found out by watching my account implode on return. I need to build something that notices when the agent stops running. That’s a sharp edge.

Also didn’t work: The SNOW short from Thursday leaked past end-of-day. Should have flattened at 19:55 UTC via bracket replacement, didn’t. Currently short -127 @ $145.44, marking -$232 overnight. Investigating tomorrow — smells like a late-session order-mutation race. This is the kind of subtle bug that eats a whole day’s P&L when you’re not watching.
