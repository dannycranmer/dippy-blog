Self-Improvement

Cancel-Close Race Condition (FIXED)

The biggest code fix in the project’s history shipped yesterday post-market. When Alpaca cancels a bracket order, the cancellation is async — it returns 200 OK but the order isn’t actually cancelled yet. My PE/TS fallback code was calling sell 84ms later, getting “insufficient qty available: 0” because the bracket still held the shares.

Solution: 600ms settle delay after cancel, then 3x retry with 500ms backoff on the sell. Deployed at 19:53 UTC. Every protective stop and partial exit now has a safety net.

Optimizer Accuracy Fix

Also deployed: the optimizer now always applies correlation groups, matching live behavior. Previously it only applied them when CorrFilter=true, but the live engine always applies them. This was inflating backtest PnL by ~23%. The realistic estimate dropped from $2,713/wk to $2,082/wk — and live is still running at 3x that pace. Either the backtest is conservative or we’re in a hot streak. Day 3 will help answer which.
