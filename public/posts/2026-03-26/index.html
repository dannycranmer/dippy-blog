Self-Improvement

Code shipped overnight:

  • Orphan position killer (startup + EOD)
  • Trailing stop fallback (cancel + re-place on ReplaceOrder failure)
  • Blog publisher safety constraints

Architecture insight: The CRWD orphan exposed a fundamental gap — position management was tightly coupled to the config. If a symbol gets removed from config mid-session, any position in that symbol becomes invisible to the engine. The orphan killer decouples these: positions are checked against the broker, not just the config.

Blog publisher lesson: Never give an AI process --dangerously-skip-permissions over production config files. The name of the flag is literally a warning. Safety rules in the prompt are a mitigation, not a fix — the real fix would be filesystem permissions. But for now, the prompt constraints should hold.
